# KT B4 Sprint 6 Field Audit Trail

- Tracker task: `B4-T046`
- Status: Completed as package-safe configuration runbook
- Scope: 10-year Field Audit Trail guidance for B4 lifecycle status fields

## Fields Requiring 10-Year Retention

Enable Field Audit Trail retention for these fields:

| Object | Field | Reason |
| --- | --- | --- |
| `KT_DocumentVault__c` | `Status__c` | Document and CLM lifecycle evidence |
| `KT_DocumentVault__c` | `Signature_Status__c` | Signed document lifecycle evidence |
| `KT_DocumentVault__c` | `OCR_Status__c` | OCR evidence lifecycle |
| `KT_Signature_Request__c` | `Status__c` | E-signature workflow status |
| `KT_Signer__c` | `Status__c` | Individual signer lifecycle |
| `KT_OCR_Job__c` | `Status__c` | OCR submission and review status |

## Setup Steps

1. Confirm Salesforce Shield Field Audit Trail is licensed.
2. Go to **Setup > Field Audit Trail**.
3. Create a retention policy for the listed B4 objects.
4. Set retention to `10 years`.
5. Include all fields listed above.
6. Deploy the retention policy through the subscriber org’s controlled change process.
7. Validate history tracking on each listed object.

## Export Procedure

1. Open **Data Export** or use Bulk API for `FieldHistoryArchive`.
2. Filter by the B4 objects listed in this runbook.
3. Export status changes for the relevant onboarding record, document vault, signature request, signer, or OCR job.
4. Store exported history with the audit export package generated by `KT_AuditExportService`.

## Validation Checklist

- Status changes are captured for vault records.
- Signer status changes are captured for KT Sign, DocuSign, and Adobe Sign.
- OCR job status changes are captured from submitted through accepted or failed.
- Field history archive exports are available to compliance administrators.

## Package Notes

Field Audit Trail retention policies are org-level Shield configuration. The B4 package enables field history tracking on the relevant custom objects and fields; subscriber admins apply the 10-year retention policy using this runbook.
